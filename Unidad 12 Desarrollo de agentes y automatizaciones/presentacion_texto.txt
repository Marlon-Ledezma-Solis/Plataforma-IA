Complementariedad: Una Sinergia Imparable
La verdadera potencia emerge cuando la automatización y la IA trabajan juntas. La automatización proporciona la estructura y la eficiencia para ejecutar tareas, mientras que la IA añade la inteligencia necesaria para optimizar y mejorar esas tareas de forma continua.
La Automatización
Crea y gestiona los flujos de trabajo repetitivos, asegurando que las operaciones básicas se realicen de forma fluida y eficiente.
La IA
Analiza los datos generados por la automatización para identificar oportunidades de mejora, optimizar decisiones y prever resultados.

--- Diapositiva 42 ---
FUNDAMENTOS
El Modelo Mental de n8n
Workflow como Grafo Dirigido
Un workflow en n8n es un grafo donde los nodos representan funciones u operaciones, las conexiones indican dependencias de ejecución, y las ramas permiten decisiones, paralelismo o rutas alternativas.
n8n no es una simple macro: es un orquestador de procesos. Cada nodo toma datos, los transforma o llama un servicio, y devuelve nuevos datos al siguiente nodo.

--- Diapositiva 43 ---
Items y JSON: La Unidad de Procesamiento
¿Qué es un Item?
La unidad básica de procesamiento en n8n. Cada item contiene típicamente JSON y opcionalmente datos binarios como archivos.
Ejemplos Prácticos
Un mensaje de Telegram = 1 item. 100 filas de una tabla = 100 items. Comprender esto evita errores comunes.
$json en Acción
Dentro de un nodo, $json se refiere al JSON del item actual que está pasando por ese nodo en ese momento.

Tipos de conexiones

ai_tool
Herramientas externas
ai_memory
Memoria contextual
AI
Conexiones especializadas
Main
Flujo normal de datos
Conexión Main
La tubería estándar que conecta Trigger → Transformaciones → Requests → Respuesta en el flujo normal de trabajo.
Conexiones Especiales de IA
Cuando usas nodos LangChain/AI, aparecen conexiones especializadas: ai_languageModel conecta el modelo, ai_memory conecta memoria, y ai_tool conecta herramientas.

--- Diapositiva 45 ---
El AI Agent funciona como un cerebro que necesita tres componentes esenciales: un motor (modelo de lenguaje), un cuaderno (memoria para recordar contexto), y manos (herramientas para ejecutar acciones).
Si conectas algo por main en vez de ai_*, el agente no lo verá. Las conexiones especializadas son fundamentales para que el sistema funcione correctamente.

--- Diapositiva 46 ---
Expresiones: El Pegamento de la Automatización
¿Qué es una Expresión?
Un pedazo de lógica dentro de un campo del nodo usando {{ ... }} que permite leer datos, generar textos dinámicos y calcular condiciones.
Usos Principales
Leer datos del item actual ($json)
Referenciar datos de otros nodos
Generar textos dinámicos
Calcular fechas y condiciones

--- Diapositiva 47 ---
Referenciar Otros Nodos
Patrón Típico
{{ $('Telegram Trigger').item.json.message.chat.id }}
Este código toma el output del nodo "Telegram Trigger", agarra el item actual, lee el JSON y extrae message.chat.id.
Regla práctica: cuando estés perdido, abre la ejecución del nodo y copia la ruta exacta del JSON. Esto te ahorrará mucho tiempo de depuración.

--- Diapositiva 48 ---
Funciones Útiles en Expresiones
{{$now}}
Obtiene la fecha y hora actual del sistema
{{$json}}
Accede al item actual que está siendo procesado
{{$env.VAR}}
Lee variables de entorno, ideal para tokens y URLs sensibles
Teoría de seguridad: Nunca hagas hardcode de tokens en nodos. Siempre usa variables de entorno o credenciales para información sensible.

--- Diapositiva 51 ---
Cómo Pensar la IA en Automatizaciones
El modelo de lenguaje redacta y decide, pero no garantiza datos precisos. Para obtener precisión real, necesitas catálogos en bases de datos, herramientas (tools) para consultar fuentes reales, y validaciones antes y después del procesamiento.
Un LLM no es un sistema de verdad absoluta: es un asistente que necesita estructura y reglas claras para funcionar correctamente en producción.

Este es el componente más importante para garantizar respuestas consistentes y profesionales.

--- Diapositiva 69 ---
Por Qué la Memoria Necesita chat.id
Explicación Didáctica
La memoria necesita una "llave" estable para funcionar correctamente. El chat.id es perfecto porque es el mismo para toda esa conversación específica.
Error común: Si el sessionKey cambia (por ejemplo, usar fecha), la memoria se "reinicia" y el bot olvida todo el contexto anterior.

--- Diapositiva 74 ---
Flujo de Conversación Esperado
El bot debe mantener una conversación natural, preguntando primero sobre necesidades, luego recomendando opciones específicas, y finalmente solicitando datos de contacto de forma secuencial (uno por uno, no todos a la vez).

--- Diapositiva 75 ---
DEBUGGING
Errores Típicos y Soluciones
❌ No llega nada al Trigger
Causas: Credencial Telegram incorrecta, token de BotFather mal copiado, o no está "listening" en test.
❌ El agente responde raro o largo
Causas: System message sin reglas claras, falta "respuestas cortas", o falta pedir datos uno por uno.
❌ Memoria no funciona
Causas: sessionKey mal puesto (debe ser chat.id), o se conectó memory por "main" en vez de "ai_memory".
❌ Telegram Send Message falla
Causas: chatId mal referenciado, o output del agente no está en $json.output (revisar panel output del nodo).

--- Diapositiva 76 ---
Depuración: Error de Data Mapping
Síntoma
Ves "undefined" en los outputs o el bot no responde correctamente.
Causa Raíz
Ruta JSON mal referenciada en las expresiones.
Solución
Abre la ejecución del nodo, copia la ruta exacta del JSON desde el panel de output, y pégala en tu expresión.

--- Diapositiva 77 ---
Depuración: Control de Flujo
Problema Común
El agente nunca se ejecuta aunque el Trigger funciona correctamente.
Causa: Un nodo IF está filtrando demasiado, bloqueando el flujo antes de que llegue al agente.
Solución: Revisa las condiciones del IF. Asegúrate de que no sean demasiado restrictivas. Prueba temporalmente sin el IF para confirmar que el agente funciona.

Asignación Correcta
Verifica que cada nodo tenga la credencial correctamente asignada
Permisos API
Asegúrate de que tu API key de OpenAI tenga créditos y permisos activos

--- Diapositiva 79 ---
Depuración: Model/Tool Wiring
Conexión por main
Herramientas conectadas incorrectamente; agente no las detecta
Conexión por ai_tool
Conexión especializada; flujo correcto y acceso a herramientas
Error Frecuente
Conectar herramientas o el modelo por la conexión "main" en lugar de las conexiones especializadas de IA.
Impacto
El agente no puede ver ni usar las herramientas conectadas incorrectamente.
Solución
Usa siempre ai_languageModel para el modelo, ai_memory para memoria, y ai_tool para herramientas.

--- Diapositiva 80 ---
Mejores Prácticas para Producción
Antes de llevar tu bot a producción, implementa validaciones robustas, manejo de errores, logging de conversaciones, y límites de rate para evitar abusos. Considera también agregar un sistema de escalamiento a humanos para casos complejos.
La seguridad debe ser prioritaria: nunca expongas tokens, usa variables de entorno, y sanitiza todas las entradas del usuario.

IF para Filtrar Mensajes
